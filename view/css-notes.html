Cascading Style Sheets

A diferencia de HTML donde las bases de aprendizaje son más claras y con una información más concreta, en CSS hay miles de opciones para personalizar y estilizar tu página. Dependiendo de cada ocasión, las ramas para solucionar un problema se dividen mucho, por lo tanto intentaré explicar los métodos más comunes y prácticos para utilizar en CSS, explicando cómo funciona.

Introducción a CSS

¿Qué es CSS?

CSS (Cascading Style Sheets, o Hojas de Estilo en Cascada) es un lenguaje de diseño utilizado para controlar la presentación y el estilo de los documentos HTML. Con CSS, puedes definir cómo se ven los elementos en una página web, como colores, fuentes, márgenes, tamaños, posiciones y más. Separa el contenido (HTML) de su presentación (diseño), lo que facilita el mantenimiento y la actualización de los estilos.

Sintaxis básica

La sintaxis de CSS se compone de:

  • Un selector: Indica a qué elemento(s) HTML se aplicarán los estilos.
  • Un bloque de declaración: Contiene una o más declaraciones entre llaves {}.
  • Declaraciones: Cada declaración incluye una propiedad y un valor, separados por dos puntos : y terminados con un punto y coma ;. 
selector {
    propiedad: valor;
}

CSS funciona mediante reglas, de las cuales se muestra un ejemplo a continuación.

h1, h3 {
    background-color: blue;
    color: white;
}

En las reglas tenemos tres cosas:

  • Los selectores. En este caso queremos modificar como van a quedar todos los <h1> y <h3>.
  • Las propiedades. En el ejemplo se pretende cambiar el color de fondo y el color de las letras.
  • Los valores. En este caso se pone el valor blue para la propiedad background-color y el valor white para la propiedad color.

En el siguiente CSS se muestran los principales selectores.


    h1{ /*Selecciona TODOS los h1 y les pone fondo azul*/
        background-color: blue;
    }
    h1, h2{ /*Selecciona TODOS LOS H1 Y TODOS LOS H2 y les pone fondo azul*/
        background-color: blue;
    }
    h1.titular{ /*Selecciona TODOS LOS H1 con el class "titular"*/
        background-color: blue;
    }
    h1#titulonoticia{ /*Selecciona SOLO UN H1, el que tenga el id indicado*/
        background-color: blue;
    }

    /* Selecciona todos los elementos li que sean hijos de un ul*/
    ul>li {
        background-color: blue;
    }

    /* Selecciona los párrafos que vayan justo detrás de un div*/
    div + p{
        background-color: blue;
    }

    /* Selecciona todos los elementos li que estén dentro de algun ul,
    INCLUSO AUNQUE NO SEAN HIJOS DIRECTOS, sino "nietos", "bisnietos"...*/
    ul li {
        background-color: blue;
    }

    /* Esto es una "pseudo-clase" selecciona los enlaces no visitados
    y los pone con fondo azul*/
    a:link{
        background-color: blue;
    }

    /* Pone en fondo rojo los enlaces ya visitados*/
    a:visited{
        background-color: blue;
}

Formas de aplicar CSS

Hay tres formas principales de aplicar CSS a un documento HTML:

  • CSS inline: Se aplica directamente a un elemento HTML usando el atributo style.
  • CSS interno: Se define dentro de una etiqueta <style> en el <head> del documento HTML.
  • CSS externo: Se escribe en un archivo separado con extensión .css y se enlaza al HTML usando la etiqueta <link>.

Ejemplos de código

  • CSS inline

Se aplica directamente a un elemento HTML usando el atributo style.

<h3 style="color: blue; font-size: 20px;">Este es un título con CSS inline</h3>
                    <p style="background-color: yellow; padding: 10px;">Este párrafo tiene un fondo amarillo y un relleno de 10px.</p>
  • CSS interno

Se define dentro de una etiqueta <style> en el <head> del documento HTML.

<head>
        <style>
                h3 {
                    color: green;
                    font-size: 24px;
                }
                p {
                    background-color: lightgray;
                    padding: 15px;
                }
        </style>
</head>

<body>
            <h3>Este es un título con CSS interno</h3>
            <p>Este párrafo tiene un fondo gris claro y un relleno de 15px.</p>
</body>
  • CSS externo

Se escribe en un archivo separado con extensión .css y se enlaza al HTML usando la etiqueta <link>.

<!DOCTYPE html>
<html lang="es">
        <head>
                <meta charset="UTF-8">
                <title>Ejemplo de CSS externo</title>
                <link rel="stylesheet" href="estilos.css">
        </head>
        <body>
                <h3>Este es un título con CSS externo</h3>
                <p>Este párrafo tiene un fondo naranja y un relleno de 20px.</p>
        </body>
</html>

Selectores

Los selectores en CSS son patrones que se utilizan para seleccionar los elementos HTML a los que se aplicarán los estilos. Hay varios tipos de selectores, desde los más básicos hasta los más avanzados.

Selectores básicos

  • Selector de elemento: Selecciona todos los elementos de un tipo específico.
    •  p {
    color: blue;
}

    Este selector aplica el color azul a todos los elementos <p>.

  • Selector de clase: Selecciona elementos con un atributo class específico. Se usa un punto (.) antes del nombre de la clase.
    • .destacado {
    background-color: yellow;
}

    Este selector aplica un fondo amarillo a todos los elementos con la clase destacado.

  • Selector de ID: Selecciona un elemento con un atributo id específico. Se usa un hashtag # antes del nombre del ID.
    • #header {
    font-size: 30px;
}

    Este selector aplica un tamaño de fuente de 30px al elemento con el ID header.

    Selectores avanzados

    Selector de pseudo-clase :hover: Aplica estilos cuando el usuario pasa el cursor sobre un elemento.

    a:hover {
                    color: red;
                }

    Este selector cambia el color del texto a rojo cuando el usuario pasa el cursor sobre un enlace.

    Selector de pseudo-clase :nth-child(): Selecciona elementos basados en su posición en un grupo de hermanos.

    li:nth-child(odd) {
                    background-color: lightgray;
                }

    Este selector aplica un fondo gris claro a los elementos <li> impares.

    Selector de hijo directo (>): Selecciona elementos que son hijos directos de otro elemento.

    ul > li {
                    list-style: none;
                }

    Este selector elimina los estilos de lista de los elementos <li> que son hijos directos de un <ul>.

    Selector de hermano adyacente (+): Selecciona un elemento que está inmediatamente después de otro.

    h1 + p {
                    font-weight: bold;
                }

    Este selector aplica negrita al primer párrafo que sigue a un <h1>.

    Selector de hermano general (~): Selecciona todos los elementos que son hermanos de otro elemento.

    h1 ~ p {
                    color: green;
                }

    Este selector aplica color verde a todos los párrafos que son hermanos de un <h1>.

    Selectores de atributos

    Selector de atributo básico: Selecciona elementos que tienen un atributo específico.

    a[target] {
                    color: purple;
                }

    Este selector aplica color púrpura a todos los enlaces que tienen el atributo target.

    Selector de atributo con valor exacto: Selecciona elementos cuyo atributo tiene un valor específico. 

    input[type="text"] {
                    border: 1px solid gray;
                }

    Este selector aplica un borde gris a todos los campos de texto (<input type="text">).

    Selector de atributo que comienza con un valor: Selecciona elementos cuyo atributo comienza con un valor específico.

    a[href^="https"] {
                    color: orange;
                }

    Este selector aplica color naranja a todos los enlaces que comienzan con https.

    Selector de atributo que termina con un valor: Selecciona elementos cuyo atributo termina con un valor específico.

    a[href$=".pdf"] {
                    color: red;
                }

    Este selector aplica color rojo a todos los enlaces que terminan con .pdf.

    Selector de atributo que contiene un valor: Selecciona elementos cuyo atributo contiene un valor específico.

    a[href*="google"] {
                    font-weight: bold;
                }

    Este selector aplica negrita a todos los enlaces que contienen la palabra google en su atributo href.

    Modelo de Caja (Box Model)

    En el diseño web, los elementos HTML tienen un modelo de caja que define cómo se estructuran visualmente. Este modelo incluye:

    1. El Modelo de Caja

    El modelo de caja está compuesto por cuatro partes principales:

    • Margen (margin): Espacio externo alrededor del elemento.
    • Borde (border): Línea que rodea el contenido y el padding.
    • Relleno (padding): Espacio entre el contenido y el borde.
    • Contenido (content): Área donde se muestra el texto o las imágenes.

    Ejemplo de Código

    
.box-example {
    width: 200px;
    height: 100px;
    margin: 20px auto; /* Margen externo */
    padding: 20px;     /* Relleno interno */
    border: 5px solid #3498db; /* Borde */
    background-color: #ecf0f1; /* Fondo del contenido */
}

    2. Ancho (width) y Alto (height)

    Los valores de width y height definen el tamaño del área de contenido.

    Sin embargo, el tamaño total del elemento incluye también el padding, el border y el margin.

    Ejemplo

    
.elemento {
    width: 300px; /* Ancho del contenido */
    height: 150px; /* Alto del contenido */
    padding: 10px; /* Añade 10px en cada lado interno */
    border: 5px solid black; /* Añade 5px en cada lado externo */
    margin: 20px; /* Añade 20px en cada lado externo */
}

    3. Propiedad box-sizing

    Por defecto, el tamaño total de un elemento incluye el contenido, el padding y el borde. Esto puede ser confuso al calcular dimensiones.

    La propiedad box-sizing permite cambiar este comportamiento:

    • content-box: El tamaño del contenido es igual a width y height.
    • border-box: El tamaño total incluye el contenido, el padding y el borde.

    Ejemplo

    
.elemento {
    box-sizing: border-box;
    width: 300px;
    padding: 20px;
    border: 10px solid black;
}

    Con box-sizing: border-box, el ancho total será siempre 300px, independientemente del padding y el borde.

    4. Propiedad overflow

    La propiedad overflow controla qué sucede cuando el contenido es demasiado grande para caber en su área asignada.

    • visible: El contenido se desborda (valor predeterminado).
    • hidden: El contenido que se desborda se oculta.
    • scroll: Se agregan barras de desplazamiento.
    • auto: Se agregan barras de desplazamiento solo si es necesario.

    Ejemplo

    
.contenedor {
    width: 200px;
    height: 100px;
    overflow: auto; /* Agrega barras de desplazamiento si es necesario */
}

    Resumen

    El modelo de caja es fundamental en CSS. Entender cómo funcionan el margin, border, padding y content te permitirá controlar mejor el diseño de tus páginas web. Además, las propiedades como width, height, box-sizing y overflow son herramientas clave para gestionar el espacio y el comportamiento de los elementos.

    Propiedades de Texto y Fuentes

    Las propiedades de texto y fuentes en CSS permiten controlar cómo se muestran los textos en una página web. Estas propiedades son esenciales para mejorar la legibilidad y el diseño visual.

    1. Propiedades Básicas de Texto

    Estas propiedades definen aspectos fundamentales del texto, como su color, tipo de fuente y tamaño.

    color

    Define el color del texto. Puede especificarse en formato hexadecimal, RGB o nombres de colores.

    
            p {
                color: #3498db; /* Azul claro */
            }

    font-family

    Especifica la fuente del texto. Se recomienda proporcionar varias opciones en caso de que la primera no esté disponible.

    
            body {
                font-family: Arial, sans-serif;
            }

    font-size

    Define el tamaño del texto. Puede expresarse en píxeles (px), ems (em) o porcentajes (%).

    
            h1 {
                font-size: 24px;
            }

    2. Propiedades de Estilo de Texto

    Estas propiedades controlan el peso, alineación y espaciado del texto.

    font-weight

    Define el grosor de la fuente. Los valores comunes son normal, bold o números como 400 (normal) y 700 (negrita).

    
            strong {
                font-weight: bold;
            }

    line-height

    Controla el espacio entre líneas de texto. Un valor mayor mejora la legibilidad.

    
            p {
                line-height: 1.6; /* Espacio relativo al tamaño del texto */
            }

    text-align

    Alinea el texto horizontalmente. Los valores comunes son left, right, center y justify. 

    
            h2 {
                text-align: center;
            }

    3. Propiedades de Decoración de Texto

    Estas propiedades añaden efectos visuales al texto, como subrayados o transformaciones.

    text-decoration

    Añade decoraciones al texto, como subrayados, tachados o líneas superiores.

    
            a {
                text-decoration: none; /* Elimina el subrayado de los enlaces */
            }

    text-transform

    Transforma el texto a mayúsculas, minúsculas o capitalización.

    
            h3 {
                text-transform: uppercase; /* Convierte el texto a mayúsculas */
            }

    4. Ejemplo Completo

    A continuación, un ejemplo que combina todas estas propiedades:

    
<style>
    .ejemplo {
        color: #e74c3c; /* Color rojo */
        font-family: 'Georgia', serif; /* Fuente serif */
        font-size: 18px; /* Tamaño de fuente */
        font-weight: bold; /* Negrita */
        line-height: 1.8; /* Espaciado entre líneas */
        text-align: center; /* Alineación centrada */
        text-decoration: underline; /* Subrayado */
        text-transform: capitalize; /* Primera letra en mayúscula */
}
</style>

    este es un ejemplo de texto con múltiples propiedades css.

    Resumen

    Las propiedades de texto y fuentes en CSS son herramientas poderosas para controlar la apariencia del contenido textual. Al dominar propiedades como color, font-family, font-size, font-weight, line-height, text-align, text-decoration y text-transform, puedes crear páginas web visualmente atractivas y fáciles de leer.

    Colores y Fondos

    Las propiedades de colores y fondos en CSS permiten controlar el aspecto visual de los elementos, desde el color del texto hasta imágenes de fondo complejas.

    1. Colores en CSS

    Los colores se pueden definir de varias formas en CSS, dependiendo de la precisión y el contexto que necesites.

    rgb

    Define un color usando valores de rojo, verde y azul (cada uno entre 0 y 255).

    
            p {
                color: rgb(255, 99, 71); /* Rojo tomate */
            }

    rgba

    Igual que rgb, pero añade un canal alfa para la transparencia (valor entre 0 y 1).

    
            div {
                background-color: rgba(0, 0, 255, 0.5); /* Azul semitransparente */
            }

    hsl

    Define un color usando matiz (hue), saturación y luminosidad. Es útil para ajustar tonos.

    
            h1 {
                color: hsl(120, 100%, 50%); /* Verde brillante */
            }

    hex

    Define un color usando un código hexadecimal de 6 dígitos (o 3 para abreviar).

    
            span {
                color: #ff5733; /* Naranja */
            }

    2. Propiedades de Fondo

    Las propiedades de fondo permiten personalizar el área detrás del contenido de un elemento.

    background-color

    Establece el color de fondo de un elemento.

    
            body {
                background-color: #f0f0f0; /* Gris claro */
            }

    background-image

    Añade una imagen como fondo de un elemento.

    
            section {
                background-image: url('imagen.jpg'); /* Ruta de la imagen */
            }

    background-size

    Controla el tamaño de la imagen de fondo. Los valores comunes son cover (ajusta la imagen al contenedor) y contain (muestra toda la imagen).

    
            div {
                background-size: cover; /* Ajusta la imagen al contenedor */
            }

    background-position

    Define la posición inicial de la imagen de fondo. Se puede usar palabras clave como top, center o coordenadas.

    
            header {
                background-position: center top; /* Centra horizontalmente y alinea arriba */
            }

    background-repeat

    Controla si la imagen de fondo se repite. Los valores comunes son repeat (repite en ambas direcciones), no-repeat (no se repite) y repeat-x/repeat-y.

    
            footer {
                background-repeat: no-repeat; /* No repite la imagen */
            }

    3. Ejemplo Completo

    A continuación, un ejemplo que combina todas estas propiedades:

    
<style>
    .ejemplo {
        background-color: #3498db; /* Color de fondo azul */
        background-image: url('paisaje.jpg'); /* Imagen de fondo */
        background-size: cover; /* Ajusta la imagen al contenedor */
        background-position: center; /* Centra la imagen */
        background-repeat: no-repeat; /* No repite la imagen */
        color: #ffffff; /* Texto blanco */
        padding: 20px;
        text-align: center;
    }
</style>

<div class="ejemplo">
    Este es un ejemplo de fondo con imagen y color.
</div>

    Resumen

    Las propiedades de colores y fondos en CSS son herramientas poderosas para mejorar el diseño visual de una página web. Al dominar propiedades como rgb, rgba, hsl, hex, background-color, background-image, background-size, background-position y background-repeat, puedes crear interfaces atractivas y funcionales.

    Diseño y Posicionamiento

    El diseño y el posicionamiento son aspectos fundamentales en el desarrollo web que determinan cómo se organizan y muestran los elementos en una página. CSS proporciona varias propiedades clave para controlar el comportamiento de los elementos, como display, position, z-index, float y clear. Estas propiedades permiten crear diseños estructurados, dinámicos y visualmente atractivos.

    ---

    Propiedad display

    La propiedad display define cómo un elemento se comporta en el flujo del documento y cómo interactúa con otros elementos. Los valores más comunes incluyen:

    • block: El elemento ocupa todo el ancho disponible y comienza en una nueva línea. Ejemplos comunes son <div>, <p> y <h1>.
    • inline: El elemento solo ocupa el espacio necesario para su contenido y no inicia una nueva línea. Ejemplos incluyen <span> y <a>.
    • inline-block: Combina las características de inline y block. El elemento se comporta como un elemento en línea pero permite establecer dimensiones (ancho y alto).
    • flex: Activa el modelo de diseño flexible (Flexbox), que permite alinear y distribuir elementos de manera eficiente en una dimensión (fila o columna).
    • grid: Activa el modelo de diseño en cuadrícula (CSS Grid), que permite organizar elementos en filas y columnas simultáneamente.

    Por ejemplo, si queremos que un botón tenga un ancho fijo y se comporte como un elemento en línea, podemos usar:

    
.button {
    display: inline-block;
    width: 150px;
}

    Esta propiedad es esencial para definir la estructura básica de un diseño y garantizar que los elementos se comporten según lo esperado.

    ---

    Propiedad position

    La propiedad position controla cómo un elemento se posiciona en relación con su contexto. Los valores más comunes son:

    • static: Es el valor predeterminado. El elemento sigue el flujo normal del documento y no se ve afectado por propiedades como top, right, bottom o left.
    • relative: El elemento se posiciona en relación con su posición original en el flujo del documento. Las propiedades top, right, bottom y left desplazan el elemento desde su posición inicial.
    • absolute: El elemento se posiciona en relación con su ancestro más cercano que tenga una posición distinta de static. Si no hay tal ancestro, se posiciona respecto al viewport.
    • fixed: El elemento se posiciona en relación con el viewport y permanece fijo incluso cuando se desplaza la página.
    • sticky: El elemento se comporta como relative hasta que alcanza un punto específico en el viewport, momento en el que se convierte en fixed.

    Ejemplo de Posicionamiento

    Supongamos que queremos crear un encabezado fijo en la parte superior de la página:

    
.header {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    background-color: #333;
    color: white;
    padding: 10px;
}

    En este caso, el encabezado permanecerá visible en la parte superior de la pantalla incluso cuando el usuario desplace la página hacia abajo.

    ---

    Propiedad z-index

    La propiedad z-index controla el orden de apilamiento de los elementos posicionados (relative, absolute, fixed o sticky). Los elementos con un valor mayor de z-index aparecen encima de los elementos con un valor menor.

    Por ejemplo:

    
.box1 {
    position: absolute;
    top: 50px;
    left: 50px;
    z-index: 1;
}

.box2 {
    position: absolute;
    top: 60px;
    left: 60px;
    z-index: 2;
}

    En este caso, box2 aparecerá encima de box1 porque tiene un valor mayor de z-index.

    ---

    Propiedades float y clear

    La propiedad float se utiliza tradicionalmente para posicionar elementos a la izquierda o derecha de su contenedor, permitiendo que el contenido fluya a su alrededor. Los valores comunes son left, right y none.

    Por ejemplo, si queremos que una imagen flote a la izquierda del texto:

    
.image {
    float: left;
    margin-right: 10px;
}

    La propiedad clear se utiliza para evitar que los elementos floten alrededor de otros elementos. Sus valores incluyen left, right, both y none.

    Por ejemplo, si queremos que un párrafo no flote junto a una imagen:

    
.paragraph {
    clear: both;
}

    Aunque float ha sido ampliamente reemplazado por Flexbox y Grid para diseños modernos, sigue siendo útil para ciertos casos específicos, como imágenes dentro de texto.

    ---

    Conclusión

    El diseño y el posicionamiento son pilares esenciales del desarrollo web que permiten controlar cómo se organizan y muestran los elementos en una página. La propiedad display define el comportamiento básico de los elementos, mientras que position permite ajustar su ubicación precisa. Además, herramientas como z-index, float y clear ofrecen un control adicional sobre la superposición y el flujo del contenido.

    Al dominar estas propiedades, puedes crear diseños estructurados, dinámicos y adaptables que funcionen correctamente en diferentes dispositivos y contextos.

    Flexbox (Diseño flexible)

    Flexbox es un modelo de diseño en CSS que permite crear interfaces flexibles y responsivas de manera más sencilla. A diferencia de los modelos tradicionales, como el flujo normal o las tablas, Flexbox ofrece una solución moderna para alinear elementos de forma precisa y adaptar su comportamiento según el espacio disponible. En este artículo, exploraremos cómo funciona Flexbox, sus propiedades principales y cómo se aplican tanto a contenedores como a elementos flexibles.

    El Contenedor Flex

    El primer paso para trabajar con Flexbox es definir un contenedor como flex. Esto se logra utilizando la propiedad display: flex; en el elemento padre. Al hacer esto, todos los elementos hijos directos del contenedor se convierten automáticamente en elementos flexibles, lo que significa que pueden ser manipulados mediante las propiedades de Flexbox.

    Una vez que el contenedor está configurado como flex, podemos controlar la disposición de los elementos hijos utilizando varias propiedades clave:

    Dirección del Eje Principal (flex-direction)

    La propiedad flex-direction define la dirección principal en la que se colocarán los elementos flexibles. Por defecto, el valor es row, lo que significa que los elementos se distribuirán horizontalmente de izquierda a derecha. Sin embargo, también podemos cambiar esta dirección a column para una disposición vertical, o incluso invertir el orden con row-reverse o column-reverse.

    
.container {
    display: flex;
    flex-direction: row; /* Opciones: row, column, row-reverse, column-reverse */
}

    Esta propiedad es fundamental porque establece el eje principal (main axis) y el eje transversal (cross axis), que son los ejes en los que se basan otras propiedades de alineación.

    Alineación Horizontal (justify-content)

    La propiedad justify-content controla cómo se distribuyen los elementos flexibles a lo largo del eje principal. Esta propiedad es especialmente útil cuando hay espacio sobrante en el contenedor y queremos decidir cómo ocuparlo. Las opciones incluyen:

    • flex-start: Los elementos se alinean al inicio del contenedor.
    • flex-end: Los elementos se alinean al final del contenedor.
    • center: Los elementos se centran en el contenedor.
    • space-between: Los elementos se distribuyen uniformemente, dejando el mismo espacio entre ellos.
    • space-around: Los elementos tienen el mismo espacio alrededor de ellos.

    Estas opciones permiten un control preciso sobre la disposición de los elementos, lo que facilita la creación de diseños equilibrados y estéticamente agradables.

    Alineación Vertical (align-items)

    Por otro lado, la propiedad align-items maneja la alineación de los elementos flexibles a lo largo del eje transversal. Al igual que justify-content, ofrece varias opciones:

    • stretch: Los elementos se estiran para ocupar todo el espacio disponible en el eje transversal (valor por defecto).
    • flex-start: Los elementos se alinean al inicio del eje transversal.
    • flex-end: Los elementos se alinean al final del eje transversal.
    • center: Los elementos se centran en el eje transversal.
    • baseline: Los elementos se alinean según su línea base.

    Esta propiedad es particularmente útil cuando queremos asegurar que los elementos tengan una alineación consistente, independientemente de su tamaño o contenido.

    Los Elementos Flexibles

    Una vez que el contenedor está configurado, los elementos flexibles dentro de él pueden ajustarse dinámicamente según las propiedades que les asignemos. Estas propiedades permiten controlar cómo los elementos crecen, se encogen o mantienen su tamaño inicial.

    Crecimiento (flex-grow)

    La propiedad flex-grow define qué tan rápido crecerá un elemento flexible en comparación con sus hermanos cuando haya espacio disponible en el contenedor. El valor predeterminado es 0, lo que significa que el elemento no crecerá. Si asignamos un valor mayor a 0, el elemento comenzará a expandirse proporcionalmente al espacio restante.

    Por ejemplo, si un elemento tiene flex-grow: 2 y otro tiene flex-grow: 1, el primero ocupará el doble de espacio que el segundo.

    Encogimiento (flex-shrink)

    De manera similar, la propiedad flex-shrink determina qué tan rápido se encogerá un elemento cuando el espacio en el contenedor sea insuficiente. El valor predeterminado es 1, lo que significa que el elemento puede reducirse. Si establecemos flex-shrink: 0, el elemento no se encogerá bajo ninguna circunstancia.

    Esta propiedad es útil para evitar que ciertos elementos se vean afectados por la falta de espacio, mientras que otros pueden ajustarse dinámicamente.

    Tamaño Base (flex-basis)

    Finalmente, la propiedad flex-basis define el tamaño inicial de un elemento flexible antes de que se apliquen las reglas de crecimiento o encogimiento. Puede expresarse en unidades absolutas (como px) o relativas (como %). El valor predeterminado es auto, lo que significa que el tamaño se basará en el contenido del elemento.

    Combinando estas tres propiedades (flex-grow, flex-shrink y flex-basis), podemos controlar completamente el comportamiento de los elementos flexibles en cualquier situación.

    
.item {
    flex-grow: 1;
    flex-shrink: 1;
    flex-basis: auto;
}

    En resumen, Flexbox proporciona un conjunto robusto de herramientas para crear diseños flexibles y adaptables. Ya sea que estés trabajando en una interfaz compleja o simplemente quieras alinear algunos elementos de manera precisa, Flexbox es una solución poderosa que simplifica el proceso de diseño web.

    CSS Grid (Diseño en cuadrícula)

    CSS Grid es una de las herramientas más poderosas para crear diseños bidimensionales en CSS. A diferencia de Flexbox, que se enfoca principalmente en disposiciones unidimensionales (ya sea filas o columnas), CSS Grid permite controlar tanto las filas como las columnas simultáneamente. Esto lo convierte en una opción ideal para diseños complejos, como maquetaciones de páginas web, dashboards o interfaces con múltiples secciones organizadas en una cuadrícula.

    En esta sección, exploraremos las propiedades clave de CSS Grid y cómo se aplican tanto al contenedor como a los elementos dentro de la cuadrícula.

    El Contenedor Grid

    Al igual que con Flexbox, el primer paso para trabajar con CSS Grid es definir un contenedor como grid. Esto se logra utilizando la propiedad display: grid; en el elemento padre. Una vez configurado, podemos definir la estructura de la cuadrícula utilizando varias propiedades específicas.

    Definición de Columnas y Filas (grid-template-columns y grid-template-rows)

    Las propiedades grid-template-columns y grid-template-rows son fundamentales para establecer la estructura básica de la cuadrícula. Estas propiedades permiten definir cuántas columnas y filas tendrá la cuadrícula, así como sus tamaños respectivos.

    Por ejemplo, si queremos una cuadrícula con tres columnas de diferentes anchos y dos filas de altura fija, podemos escribir:

    
    .container {
        display: grid;
        grid-template-columns: 100px 200px auto; /* Tres columnas */
        grid-template-rows: 50px 100px; /* Dos filas */
    }

    En este caso, la primera columna tiene un ancho fijo de 100px, la segunda de 200px y la tercera se ajustará automáticamente al espacio restante. Las filas tienen alturas fijas de 50px y 100px, respectivamente. Este enfoque proporciona un control preciso sobre la disposición de los elementos.

    Espaciado entre Elementos (gap)

    La propiedad gap (también conocida como grid-gap en versiones anteriores) define el espacio entre las filas y las columnas de la cuadrícula. Esta propiedad es especialmente útil para evitar que los elementos se superpongan o queden demasiado juntos, mejorando la legibilidad y el diseño visual.

    Podemos especificar un único valor para aplicar el mismo espaciado tanto a filas como a columnas, o bien definir valores separados para cada uno:

    
    .container {
        gap: 20px; /* Espacio uniforme entre filas y columnas */
        /* O también */
        gap: 10px 30px; /* 10px entre filas, 30px entre columnas */
    }

    Esta propiedad simplifica el proceso de añadir márgenes consistentes entre los elementos de la cuadrícula sin necesidad de usar márgenes individuales en cada elemento.

    Áreas de Cuadrícula (grid-area y grid-template-areas)

    Una de las características más poderosas de CSS Grid es la capacidad de definir áreas de cuadrícula mediante nombres descriptivos. Esto se logra utilizando las propiedades grid-template-areas y grid-area.

    La propiedad grid-template-areas permite crear un mapa visual de la cuadrícula, donde cada área tiene un nombre asignado. Luego, podemos asignar estos nombres a los elementos individuales usando grid-area.

    Por ejemplo, supongamos que queremos crear un diseño básico con un encabezado, un pie de página, una barra lateral y un área de contenido principal:

    
    .container {
        display: grid;
        grid-template-areas:
            "header header"
            "sidebar main"
            "footer footer";
        grid-template-columns: 200px auto; /* Dos columnas */
        grid-template-rows: 100px auto 50px; /* Tres filas */
    }

    .header {
        grid-area: header;
    }

    .sidebar {
        grid-area: sidebar;
    }

    .main {
        grid-area: main;
    }

    .footer {
        grid-area: footer;
    }

    Este enfoque facilita la creación de diseños complejos de manera intuitiva, ya que podemos visualizar la estructura de la cuadrícula directamente en el código.

    Alineación de Elementos

    Además de definir la estructura de la cuadrícula, CSS Grid ofrece propiedades para controlar la alineación de los elementos dentro de las celdas. Estas propiedades son similares a las de Flexbox, pero están adaptadas al contexto bidimensional de las cuadrículas.

    Alineación Horizontal (justify-items)

    La propiedad justify-items controla la alineación horizontal de los elementos dentro de sus celdas. Las opciones incluyen:

    • start: Los elementos se alinean al inicio de la celda.
    • end: Los elementos se alinean al final de la celda.
    • center: Los elementos se centran horizontalmente.
    • stretch: Los elementos se estiran para ocupar todo el ancho de la celda (valor por defecto).

    Esta propiedad es útil cuando queremos asegurar que los elementos tengan una alineación consistente en todas las columnas de la cuadrícula.

    Alineación Vertical (align-items)

    De manera similar, la propiedad align-items maneja la alineación vertical de los elementos dentro de sus celdas. Las opciones son idénticas a las de justify-items, pero actúan en el eje vertical:

    • start: Los elementos se alinean en la parte superior de la celda.
    • end: Los elementos se alinean en la parte inferior de la celda.
    • center: Los elementos se centran verticalmente.
    • stretch: Los elementos se estiran para ocupar toda la altura de la celda (valor por defecto).

    Estas propiedades permiten un control preciso sobre la posición de los elementos dentro de las celdas, lo que es especialmente útil para crear diseños equilibrados y visualmente atractivos.

    Conclusión

    CSS Grid es una herramienta versátil y potente para crear diseños bidimensionales complejos. Con su capacidad para definir estructuras de cuadrícula precisas, gestionar el espaciado entre elementos y controlar la alineación de manera intuitiva, Grid se ha convertido en una solución indispensable para el diseño web moderno. Ya sea que estés trabajando en una página simple o en una interfaz compleja, CSS Grid ofrece las herramientas necesarias para implementar tus ideas de manera eficiente y efectiva.

    Animaciones y Transiciones

    Las animaciones y transiciones son herramientas esenciales en CSS para añadir dinamismo y mejorar la experiencia del usuario en una página web. Aunque ambas técnicas se utilizan para crear efectos visuales, tienen propósitos y niveles de complejidad diferentes. Las transiciones son ideales para cambios simples y rápidos, mientras que las animaciones permiten crear efectos más elaborados y controlados.

    A continuación, exploraremos cómo funcionan las transiciones y las animaciones, sus propiedades principales y cómo aplicarlas en tus proyectos.

    Transiciones (transition)

    Las transiciones permiten cambiar suavemente el valor de una propiedad CSS durante un período de tiempo específico. Son útiles para crear efectos como cambios de color, desplazamientos o ajustes de tamaño cuando un usuario interactúa con un elemento (por ejemplo, al pasar el cursor sobre él).

    La propiedad transition es una forma abreviada que combina varias subpropiedades:

    • transition-property: Especifica qué propiedad CSS se animará.
    • transition-duration: Define cuánto tiempo durará la transición.
    • transition-timing-function: Controla la velocidad de la transición a lo largo del tiempo.
    • transition-delay: Indica cuánto tiempo debe esperarse antes de que comience la transición.

    Propiedad Animada (transition-property)

    La propiedad transition-property define qué propiedad CSS será afectada por la transición. Puede ser cualquier propiedad animable, como color, background-color, width, opacity, etc. También puedes usar el valor all para aplicar la transición a todas las propiedades animables del elemento.

    Duración de la Transición (transition-duration)

    La propiedad transition-duration especifica cuánto tiempo durará la transición. Se expresa en segundos (s) o milisegundos (ms). Por ejemplo, un valor de 0.5s indica que la transición tardará medio segundo en completarse.

    Función de Tiempo (transition-timing-function)

    La propiedad transition-timing-function determina cómo progresa la transición a lo largo del tiempo. Algunas opciones comunes incluyen:

    • ease: Comienza lentamente, acelera y luego desacelera nuevamente (valor predeterminado).
    • linear: Progresa a una velocidad constante.
    • ease-in: Comienza lentamente y luego acelera.
    • ease-out: Comienza rápidamente y luego desacelera.
    • ease-in-out: Combina ease-in y ease-out.

    Ejemplo de Transición

    Supongamos que queremos crear un botón que cambie de color suavemente cuando el usuario pase el cursor sobre él. Podríamos escribir:

    
.button {
    background-color: blue;
    transition: background-color 0.5s ease;
}

.button:hover {
    background-color: red;
}

    En este ejemplo, la propiedad background-color cambia de azul a rojo durante medio segundo con una función de tiempo ease. El resultado es un efecto visual suave y atractivo.

    Animaciones (@keyframes y animation)

    Mientras que las transiciones son ideales para cambios simples, las animaciones permiten crear efectos más complejos y personalizados. Esto se logra mediante la combinación de la regla @keyframes y la propiedad animation.

    Definición de Animaciones con @keyframes

    La regla @keyframes define los pasos intermedios de una animación. Permite especificar cómo deben cambiar las propiedades CSS en diferentes puntos del tiempo. Cada punto se define como un porcentaje, donde 0% representa el inicio y 100% el final.

    Por ejemplo, si queremos que un cuadro cambie de color gradualmente, podríamos escribir:

    
@keyframes changeColor {
    0% {
        background-color: blue;
    }
    50% {
        background-color: green;
    }
    100% {
        background-color: red;
    }
}

    Este conjunto de reglas define una animación llamada changeColor que cambia el color de fondo de azul a verde y luego a rojo.

    Aplicación de Animaciones con animation

    Una vez definida la animación con @keyframes, podemos aplicarla a un elemento utilizando la propiedad animation. Esta propiedad es una forma abreviada que combina varias subpropiedades:

    • animation-name: Especifica el nombre de la animación definida con @keyframes.
    • animation-duration: Define cuánto tiempo durará la animación.
    • animation-timing-function: Controla la velocidad de la animación a lo largo del tiempo.
    • animation-delay: Indica cuánto tiempo debe esperarse antes de que comience la animación.
    • animation-iteration-count: Determina cuántas veces se repetirá la animación (puede ser un número o infinite).
    • animation-direction: Especifica si la animación debe reproducirse hacia adelante, hacia atrás o alternar entre ambos.

    Ejemplo de Animación

    Supongamos que queremos que un cuadro realice una animación infinita de cambio de color. Podríamos escribir:

    
.box {
    width: 100px;
    height: 100px;
    animation: changeColor 5s infinite alternate;
}

@keyframes changeColor {
    0% {
        background-color: blue;
    }
    100% {
        background-color: red;
    }
}

    En este ejemplo, el cuadro cambiará su color de fondo de azul a rojo y viceversa durante 5 segundos, repitiendo la animación indefinidamente (infinite) y alternando la dirección (alternate).

    Conclusión

    Las transiciones y animaciones son herramientas poderosas para mejorar la interactividad y el atractivo visual de una página web. Mientras que las transiciones son ideales para cambios simples y rápidos, las animaciones permiten crear efectos más elaborados y personalizados. Al dominar estas técnicas, puedes crear experiencias de usuario más dinámicas y atractivas, mejorando significativamente la calidad de tus diseños web.

    Media Queries y Diseño Responsivo

    El diseño responsivo es una técnica fundamental en el desarrollo web moderno que permite que una página web se adapte a diferentes tamaños de pantalla, desde dispositivos móviles hasta pantallas de escritorio. Las media queries son la herramienta principal para implementar este tipo de diseño, ya que permiten aplicar estilos específicos según las características del dispositivo, como su ancho, alto o resolución.

    En esta sección, exploraremos cómo funcionan las media queries, los breakpoints más comunes y las unidades relativas que complementan el diseño responsivo.

    Media Queries (@media)

    Las media queries son reglas CSS que permiten aplicar estilos condicionales basados en características específicas del dispositivo, como su ancho de pantalla, orientación o densidad de píxeles. La sintaxis básica de una media query es:

    
@media (condición) {
    /* Estilos específicos */
}

    La condición puede incluir propiedades como max-width, min-width, orientation, entre otras. Esto permite ajustar el diseño de manera dinámica según el contexto del usuario.

    Breakpoints Comunes

    Los breakpoints son puntos específicos en los que el diseño cambia para adaptarse a diferentes tamaños de pantalla. Aunque no hay reglas estrictas sobre qué valores usar, existen algunos breakpoints comunes que se utilizan para categorizar dispositivos:

    • Móviles pequeños: Hasta 480px (max-width: 480px)
    • Móviles grandes y tablets pequeñas: Entre 481px y 768px (max-width: 768px)
    • Tablets y pantallas pequeñas de escritorio: Entre 769px y 1024px (max-width: 1024px)
    • Pantallas de escritorio: Más de 1025px (min-width: 1025px)

    Estos valores son solo una guía general y pueden ajustarse según las necesidades del proyecto. Lo importante es que los breakpoints se definan en función del contenido y no de dispositivos específicos.

    Ejemplo de Media Query

    Supongamos que queremos cambiar el tamaño de un encabezado cuando la pantalla sea menor a 768px. Podríamos escribir:

    
h1 {
    font-size: 2rem;
}

@media (max-width: 768px) {
    h1 {
        font-size: 1.5rem;
    }
}

    En este ejemplo, el tamaño del encabezado se reduce cuando la pantalla tiene un ancho máximo de 768px, lo que mejora la legibilidad en dispositivos más pequeños.

    Propiedades max-width y min-width

    Las propiedades max-width y min-width son las más utilizadas en las media queries para definir rangos de ancho de pantalla. Aquí está la diferencia entre ambas:

    • max-width: Aplica estilos cuando el ancho de la pantalla es igual o menor al valor especificado.
    • min-width: Aplica estilos cuando el ancho de la pantalla es igual o mayor al valor especificado.

    Por ejemplo, si queremos aplicar estilos solo a pantallas con un ancho entre 768px y 1024px, podríamos combinar ambas propiedades:

    
@media (min-width: 768px) and (max-width: 1024px) {
    /* Estilos para tablets */
}

    Esta combinación asegura que los estilos solo se apliquen dentro de ese rango específico.

    Unidades Relativas

    Además de las media queries, el uso de unidades relativas es clave para crear diseños responsivos. Estas unidades permiten que los elementos se ajusten dinámicamente según el tamaño de la pantalla o el contexto del usuario.

    em y rem

    Las unidades em y rem son relativas al tamaño de fuente. La diferencia entre ambas es que em es relativa al tamaño de fuente del elemento padre, mientras que rem es relativa al tamaño de fuente del elemento raíz (html).

    Por ejemplo, si el tamaño de fuente del elemento raíz es 16px:

    • 1rem equivale a 16px.
    • 2rem equivale a 32px.

    El uso de rem es especialmente útil para mantener una escala consistente en todo el diseño.

    %

    La unidad % define el tamaño de un elemento en relación con su contenedor padre. Por ejemplo, si un contenedor tiene un ancho de 500px y un elemento hijo tiene un ancho del 50%, este último tendrá un ancho de 250px.

    Esta unidad es ideal para crear diseños fluidos que se ajusten al tamaño del contenedor.

    vw y vh

    Las unidades vw (viewport width) y vh (viewport height) son relativas al tamaño de la ventana del navegador. Un valor de 1vw equivale al 1% del ancho de la ventana, mientras que 1vh equivale al 1% de su altura.

    Por ejemplo:

    • 50vw: Ocupa el 50% del ancho de la ventana.
    • 100vh: Ocupa el 100% de la altura de la ventana.

    Estas unidades son útiles para crear diseños que ocupen todo el espacio disponible en la pantalla.

    Conclusión

    El diseño responsivo es esencial para garantizar que una página web funcione correctamente en todos los dispositivos. Las media queries permiten adaptar el diseño según el tamaño de la pantalla, mientras que las unidades relativas aseguran que los elementos se ajusten dinámicamente al contexto del usuario.

    Al combinar estas herramientas, puedes crear experiencias web fluidas y accesibles que se adapten a cualquier dispositivo, mejorando tanto la usabilidad como la satisfacción del usuario.

    Variables en CSS y Preprocesadores

    El uso de variables en CSS y preprocesadores como SASS y LESS es una práctica fundamental para escribir código más limpio, mantenible y reutilizable. Las variables permiten almacenar valores que se pueden usar repetidamente en un proyecto, lo que facilita la gestión de estilos globales y la implementación de cambios consistentes.

    A continuación, exploraremos cómo funcionan las variables nativas en CSS y cómo los preprocesadores como SASS y LESS amplían estas capacidades con características adicionales.

    ---

    Variables en CSS

    Las variables nativas en CSS son una característica introducida en CSS3 que permite definir valores reutilizables utilizando la sintaxis --nombre-variable. Estas variables se declaran dentro de un selector y se utilizan con la función var(--nombre-variable).

    Declaración de Variables

    Para declarar una variable en CSS, se utiliza la sintaxis --nombre-variable: valor;. Las variables deben declararse dentro de un selector, generalmente en el selector :root para hacerlas accesibles globalmente en todo el documento.

    
:root {
    --color-primario: #3498db;
    --fuente-principal: 'Arial', sans-serif;
    --espaciado: 1rem;
}

    En este ejemplo, hemos definido tres variables globales: un color primario, una fuente principal y un espaciado base. Estas variables pueden ser utilizadas en cualquier parte del documento.

    Uso de Variables

    Para usar una variable, se utiliza la función var(--nombre-variable). Por ejemplo:

    
body {
    background-color: var(--color-primario);
    font-family: var(--fuente-principal);
    margin: var(--espaciado);
}

.button {
    padding: var(--espaciado);
    background-color: var(--color-primario);
}

    Este enfoque simplifica la gestión de estilos, ya que si necesitamos cambiar el color primario o la fuente principal, solo tenemos que actualizar la variable en un solo lugar.

    Ventajas de las Variables en CSS

    • Reutilización: Evita la duplicación de valores en el código.
    • Mantenibilidad: Facilita la implementación de cambios globales.
    • Consistencia: Asegura que los mismos valores se utilicen en todo el proyecto.
    ---

    Introducción a Preprocesadores: SASS y LESS

    Los preprocesadores como SASS (Syntactically Awesome Style Sheets) y LESS son herramientas que extienden las capacidades de CSS al agregar características avanzadas como variables, funciones, mixins y anidamiento. Estos preprocesadores generan CSS estándar que puede ser interpretado por los navegadores.

    Variables en SASS y LESS

    Tanto SASS como LESS permiten declarar variables de manera similar a CSS, pero con algunas diferencias en la sintaxis. En SASS, las variables se declaran con el símbolo $, mientras que en LESS se usan dos guiones bajos @.

    Ejemplo en SASS

    
$color-primario: #3498db;
$fuente-principal: 'Arial', sans-serif;
$espaciado: 1rem;

body {
    background-color: $color-primario;
    font-family: $fuente-principal;
    margin: $espaciado;
}

    Ejemplo en LESS

    
@color-primario: #3498db;
@fuente-principal: 'Arial', sans-serif;
@espaciado: 1rem;

body {
    background-color: @color-primario;
    font-family: @fuente-principal;
    margin: @espaciado;
}

    Al igual que en CSS, estas variables pueden ser reutilizadas en todo el proyecto, pero los preprocesadores ofrecen funcionalidades adicionales que las hacen aún más poderosas.

    Otras Características de SASS y LESS

    Además de las variables, SASS y LESS incluyen características avanzadas que facilitan el desarrollo de estilos complejos:

    • Anidamiento: Permite escribir selectores CSS de manera jerárquica, lo que mejora la legibilidad del código.
    • Mixins: Son bloques de código reutilizables que pueden incluir múltiples propiedades y reglas. Los mixins son útiles para evitar la repetición de código.
    • Funciones: Permiten realizar operaciones matemáticas, manipular colores y trabajar con valores dinámicos.
    • Importación de Archivos: Facilita la organización del código al dividirlo en múltiples archivos que se pueden importar en un archivo principal.

    Ejemplo de Anidamiento en SASS

    
.navbar {
    background-color: $color-primario;

    ul {
        list-style: none;

        li {
            display: inline-block;
            margin-right: $espaciado;
        }
    }
}

    Este ejemplo muestra cómo el anidamiento en SASS simplifica la escritura de selectores complejos, mejorando la estructura del código.

    Ventajas de Usar Preprocesadores

    • Mayor Flexibilidad: Ofrecen características avanzadas que no están disponibles en CSS puro.
    • Mejor Organización: Facilita la división del código en múltiples archivos y su posterior compilación.
    • Productividad: Reduce la cantidad de código repetitivo y acelera el proceso de desarrollo.
    ---

    Conclusión

    Las variables en CSS y los preprocesadores como SASS y LESS son herramientas esenciales para el desarrollo web moderno. Mientras que las variables nativas en CSS proporcionan una forma sencilla de gestionar valores reutilizables, los preprocesadores amplían estas capacidades con características avanzadas que mejoran la productividad y la organización del código.

    Al dominar estas herramientas, puedes crear proyectos más eficientes, mantenibles y escalables, asegurando que tus estilos sean consistentes y adaptables a futuros cambios.
